{% comment %} Bloco de confirmação de uma ação de fluxo da requisição (autorizar, recusar, cancelar, estornar, retornar). Cada instância é autocontida: título + descrição + botão de disparo + modal de confirmação, envolvidos no mesmo x-data="modalController(...)". Parâmetros: layout "card" (cartão dentro do grid de decisão, renderizado via components/alert.html) ou "banner" (seção completa, título+descrição à esquerda e botão à direita — usado por cancelamento e estorno; renderizado pelo layout="row" de components/alert.html) variant_token token de cor (primary | warning | danger) repassado como variant a components/alert.html — "primary" cai no ramo `{% else %}` do alert.html (mesmas classes de "info") modal_id id do modal (usado em x-data, data-modal-trigger e no id do components/modal.html) abrir_ao_carregar_expr (opcional) expressão já resolvida (ex.: `erro|yesno:"true,false"`) para abrirAoCarregar no x-data heading_id (obrigatório em layout="banner") id do h2 para aria-labelledby da section titulo texto do título (h3 no card, h2 no banner) conteudo texto da descrição desc_class (obrigatório em layout="banner") classe completa do parágrafo de descrição bg_class (opcional, só layout="banner") classe de fundo da section — default bg-{variant_token}-subtle botao_label texto do botão de disparo botao_variant variant repassado a components/button.html botao_icon (opcional) icon_template repassado a components/button.html botao_class (opcional, só layout="banner") class extra repassada a components/button.html (ex.: "shrink-0") botao_aria_haspopup (opcional, só layout="banner") aria_haspopup repassado a components/button.html (ex.: "dialog") Os parâmetros abaixo são repassados diretamente a components/modal.html: modal_titulo, modal_descricao, action_url, confirm_label, confirm_variant, icon_variant, form_body_template, hidden_inputs, erro, motivo_recusa, form Uso (card): {% include "requisicoes/partials/_confirmacao_acao.html" with layout="card" variant_token="primary"
