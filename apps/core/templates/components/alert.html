{% comment %} Componente global de caixa de aviso (banner estático). Não conhece exceções de domínio — só variante visual e role ARIA: warning/danger=alert, success/info=status (equivalente ao contrato error/warning=alert do messages-contract; "danger" é o nome de variante aqui, análogo a button.html — ver docs/plans/77-alert-component.md). Fora do contrato de dismiss de docs/CONVENTIONS.md (§Níveis e ARIA) — essa regra governa o contrato de flash messages Django, renderizado por core/partials/_messages.html, que não usa este componente. Parâmetros: variant (opcional, default=info) info, success, warning, danger message (obrigatório, exceto se body_template ou casca JS-hidratada) texto autoescapado (Django escapa por padrão). Conteúdo HTML rico só via body_template. Se body_template também for passado, body_template tem precedência e message é ignorado. body_template (opcional) caminho de partial incluído no corpo — herda o contexto do chamador, como form_body_template em modal.html. icone (opcional, boolean, default=True) — sem efeito em layout="row" role (opcional) sobrescreve o role padrão (ex. "note"). Em layout="row" não há default automático — só emite role se explicitamente passado. aria_live (opcional) valor de aria-live — sem default automático id (opcional) — necessário para cascas hidratadas por JS do chamador via getElementById (ex. caixa de duplicidade) class (opcional) passthrough para ajuste de layout do chamador bg_class (opcional) sobrescreve a classe de fundo padrão da variant (ex. para opacidade customizada) layout (opcional, default="stack") "stack": ícone (opcional) + conteúdo empilhado, banner estático de mensagem. "row": conteúdo à esquerda + slot de ação à direita — cartão/seção de confirmação com um controle persistente (ex. botão que abre um modal), sem ícone e sem role automático.
